Todas las solicitudes a la API de PrimeNet deben autenticarse mediante un **API Key** y deben incluir el ID del tenant al cual pertenece el key. Si aún no tienes un API Key, consulta la guía [Generar un API Key](/generar-api-key).

# API Key

## Formato del API Key

Un API Key de PrimeNet tiene el siguiente formato:

```text
pnk_{keyId}.{secret}
```

- **`pnk_`**: prefijo fijo que identifica el token como una llave de PrimeNet.
- **`keyId`**: identificador público de la llave, usado para localizarla.
- **`secret`**: componente secreto usado para verificar la autenticidad de la solicitud.

<Warning>
  Trata tu API Key como una contraseña. No lo compartas, no lo incluyas en repositorios públicos ni en código del lado del cliente.
</Warning>

## Scopes y permisos

Cada API Key tiene uno o más **scopes** asignados en el momento de su creación. Estos scopes determinan qué recursos y acciones puede ejecutar la integración. Si intentas acceder a un recurso para el cual tu API Key no tiene el scope necesario, la API responderá con `403 Forbidden`.

## Vencimiento y revocación

Si tu API Key tiene una fecha de vencimiento configurada, dejará de funcionar automáticamente a partir de esa fecha; todas las solicitudes posteriores recibirán `401 Unauthorized`.

Un administrador puede revocar un API Key en cualquier momento desde **Configuración → Integración de Terceros**. Una vez revocado, el key deja de funcionar de inmediato y no puede reactivarse.

<Tip>
  Si necesitas rotar tu API Key de forma periódica por seguridad, la práctica recomendada es crear un nuevo registro, actualizar tu integración para usar el nuevo token, y luego revocar el anterior.
</Tip>

# Encabezados requeridos

Cada solicitud debe incluir dos encabezados:

| Encabezado | Descripción |
| --- | --- |
| `X-Api-Key` | Tu API Key, en formato `pnk_{keyId}.{secret}` |
| `X-Tenant-Id` | El identificador del tenant al cual pertenece el key |

<Note>
  El encabezado `X-Api-Key` es lo que determina que la solicitud se autentique como una integración de terceros.
</Note>

<Warning>
  Si `X-Tenant-Id` no corresponde al tenant asociado al API Key, la solicitud será rechazada con `401 Unauthorized`.
</Warning>

<CodeGroup>

```bash cURL
curl https://api.primenetcloud.com/v1/associates \
  -H "X-Api-Key: pnk_abc123.s3cr3tValue" \
  -H "X-Tenant-Id: e3d5e6af-3ec2-49c1-92bd-9632171e9a66"
```

```javascript Node.js
const response = await fetch("https://api.primenetcloud.com/v1/associates", {
  headers: {
    "X-Api-Key": "pnk_abc123.s3cr3tValue",
    "X-Tenant-Id": "e3d5e6af-3ec2-49c1-92bd-9632171e9a66"
  }
});
```

```python Python
import requests

response = requests.get(
    "https://api.primenetcloud.com/v1/associates",
    headers={
        "X-Api-Key": "pnk_abc123.s3cr3tValue",
        "X-Tenant-Id": "e3d5e6af-3ec2-49c1-92bd-9632171e9a66"
    }
)
```

</CodeGroup>

# Errores de autenticación

<ResponseField name="401 Unauthorized">
  El API Key no fue enviado, no existe, fue revocado, expiró según su fecha de vencimiento, o el `X-Tenant-Id` no corresponde al tenant asociado al API Key.
</ResponseField>

<ResponseField name="403 Forbidden">
  El API Key es válido, pero no tiene el scope necesario para realizar la acción solicitada.
</ResponseField>
