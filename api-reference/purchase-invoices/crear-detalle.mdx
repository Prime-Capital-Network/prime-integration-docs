---
title: "Crear Detalle"
---

Crea un nuevo detalle para una factura de compra específica.

```http
POST /purchase-invoices/{id}/details
```

<Note>
  Este endpoint requiere el scope **`PurchaseInvoices.Write`**.
</Note>

### Path parameters

<ParamField path="id" type="integer" required>
  Identificador único de la factura de compra a la que se agregará el detalle.
</ParamField>

### Body parameters

<ParamField body="itemType" type="string" required>
  Tipo de ítem a registrar. Valores aceptados:

  - `Charge` (Cargo)
  - `FixedAsset` (Activo Fijo)
  - `Product` (Producto)
  - `ChartOfAccount` (Cuenta Contable)
</ParamField>

<ParamField body="itemId" type="integer" required>
  Identificador del ítem asociado. 
  
  - Si `itemType` es `Charge`
    - Debe ser un valor numérico válido del tipo `short`.
    - Debe ser un cargo con la cuenta de gasto (`ExpensAccount`) asignado.
</ParamField>

<ParamField body="chargeTypeId" type="integer">
  Identificador del tipo de cargo.

  - Este campo debe ser `null` si el campo `itemType` no es `Charge`.
  - Si la factura tiene asignado el campo `masterId`, y el master tiene el campo `transportType` igual a `Rail`, este campo no puede ser igual a `1` (Prorrateo Por Peso (WM)).

  Valores aceptados:
  - `1`	(Prorrateo Por Peso (WM))
  - `2`	(Ganancias Agentes)
  - `3`	(Distribucion En Montos Equitativos)
  - `4`	(Impuestos)
  - `5`	(Prorrateo Por Peso Fisico)
  - `6`	(Prorrateo Por Volumen)
</ParamField>

<ParamField body="isTransparent" type="boolean" default="false">
  Indica si el cargo es transparente.
</ParamField>

<ParamField body="taxPercent" type="decimal">
  Porcentaje de impuesto a aplicar. Debe ser mayor o igual a cero. Precisión de hasta 2 decimales.
</ParamField>

<ParamField body="description" type="string">
  Descripción del detalle. Máximo 250 caracteres.
</ParamField>

<ParamField body="calculationTypeId" type="string">
  Identificador del tipo de cálculo.
</ParamField>

<ParamField body="quantity" type="decimal" required>
  Cantidad del ítem. Debe ser mayor que cero. Precisión de hasta 4 decimales.
</ParamField>

<ParamField body="productPackagingId" type="integer">
  Identificador del empaque del producto.
</ParamField>

<ParamField body="packagingQuantity" type="decimal" default="1">
  Cantidad de empaques. Debe ser mayor que cero. Precisión de hasta 4 decimales.
</ParamField>

<ParamField body="price" type="decimal" required>
  Precio unitario. Debe ser mayor que cero. Precisión de hasta 4 decimales.
</ParamField>

<ParamField body="discount" type="decimal">
  Monto de descuento aplicado. Debe ser mayor o igual a cero.
</ParamField>

<ParamField body="goodServiceApplicability" type="string">
  Aplicabilidad de bienes y servicios.
</ParamField>

<ParamField body="operationCategory" type="string">
  Categoría de la operación logística asociada. Requerido si `operationId` está presente. Valores aceptados:

  - `Shipment` (Embarque)
  - `CustomManagement` (Aduanas)
  - `Additional` (Misceláneos)
  - `InlandFreight` (Transporte Terrestre)
</ParamField>

<ParamField body="operationId" type="integer">
  Identificador de la operación logística asociada. Requerido si `operationCategory` está presente.
</ParamField>

<ParamField body="departmentId" type="integer">
  Identificador del departamento asignado (Dimensión).

  - Debe ser una dimensión con el campo `typeId` igual a `1` (Department).
</ParamField>

<ParamField body="costCenterId" type="integer">
  Identificador del centro de costo asignado (Dimensión).

  - Debe ser una dimensión con el campo `typeId` igual a `2` (CostCenter).
</ParamField>

<ParamField body="projectId" type="integer">
  Identificador del proyecto asignado (Dimensión).

  - Debe ser una dimensión con el campo `typeId` igual a `3` (Project).
</ParamField>

<ParamField body="estimateId" type="integer">
  Identificador del estimado de gasto (Service Expense) asociado.
</ParamField>

<ParamField body="acquiredWarranty" type="string">
  Fecha de adquisición de la garantía en formato ISO 8601.

  - Este campo debe ser `null` si el campo `itemType` no es `FixedAsset`.
  - Esta fecha debe ser mayor o igual al campo de fecha (`date`) de la factura.
</ParamField>

### Example request

<CodeGroup>

```bash curl
curl --request POST \
  --url "https://api.primenetcloud.com/v1/purchase-invoices/1054/details" \
  --header "X-Api-Key: YOUR_API_KEY" \
  --header "X-Tenant-Id: TENANT_ID" \
  --header "Content-Type: application/json" \
  --data '{
    "itemType": "Charge",
    "itemId": 5,
    "chargeTypeId": 2,
    "isTransparent": false,
    "taxPercent": 18.00,
    "description": "Flete marítimo principal",
    "calculationTypeId": "BLS",
    "quantity": 1,
    "price": 1500,
    "discount": 0,
    "operationId": 5321,
    "operationCategory": "Shipment"
  }'
```

```python Python
        "taxPercent": 18.00,
        "description": "Flete marítimo principal",
        "calculationTypeId": "BLS",

        "price": 1500,
        "discount": 0,
        "operationId": 5321,
        "operationCategory": "Shipment"
    },
    headers={
        "X-Api-Key": "YOUR_API_KEY",
        "X-Tenant-Id": "TENANT_ID",
    },
)

print(response.json())
```

### Example response

```json
{
  "id": 1
}
```

### Response fields

<ResponseField name="id" type="integer">
  Identificador único del detalle de factura creado.
</ResponseField>

### Errores

<ResponseField name="403 Forbidden">
  El API Key no tiene el scope `PurchaseInvoices.Write` requerido.
</ResponseField>

<ResponseField name="404 Not Found">
  No existe ninguna factura de compra con el `id` especificado en la URL.
</ResponseField>
