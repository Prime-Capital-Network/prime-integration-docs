Crea un nuevo estimado de gasto (operation expense) en la empresa.

```http
POST /operation-expenses
```

<Note>
  Este endpoint requiere el scope **`OperationExpenses.Write`**.
</Note>

### Body parameters

<ParamField body="chargeId" type="integer" required>
  Identificador del cargo asociado al gasto.
</ParamField>

<ParamField body="chargeDescription" type="string" required>
  Descripción del cargo. Máximo 120 caracteres.
</ParamField>

<ParamField body="chargeDetail" type="string">
  Detalle adicional del cargo. Máximo 250 caracteres.
</ParamField>

<ParamField body="chargeTypeId" type="integer" required>
  Identificador del tipo de cargo.
</ParamField>

<ParamField body="quotationGroupId" type="integer" required>
  Identificador del grupo de cotización asociado.
</ParamField>

<ParamField body="calculationTypeId" type="string">
  Identificador del tipo de cálculo.
</ParamField>

<ParamField body="currencyId" type="string" required>
  Identificador de la moneda.
</ParamField>

<ParamField body="quantity" type="decimal" required>
  Cantidad. Debe ser mayor que cero, y menor o igual a 9,999,999.
</ParamField>

<ParamField body="price" type="decimal" required>
  Precio unitario. Debe ser mayor que cero, y menor o igual a 99,999,999.
</ParamField>

<ParamField body="taxPercent" type="decimal" required>
  Porcentaje de impuesto. Debe ser mayor o igual a cero.
</ParamField>

<ParamField body="discount" type="decimal" required>
  Descuento aplicado. Debe ser mayor o igual a cero, y menor o igual a 99,999,999.
</ParamField>

<ParamField body="vendorId" type="integer">
  Identificador del vendor asociado al gasto.
</ParamField>

<ParamField body="operationId" type="integer">
  Identificador de la operación logística asociada (house, custom, miscellaneous o ground transportation, según `operationCategory`).

  - Requerido si `operationCategory` está presente, y viceversa.
  - Debe estar presente `operationId`/`operationCategory` o `masterId`; al menos uno de los dos es requerido.
</ParamField>

<ParamField body="operationCategory" type="string">
  Categoría de la operación logística asociada.
  
  - Requerido si `operationId` está presente, y viceversa.

  Valores aceptados:

  - `Shipment` (Embarque)
  - `CustomManagement` (Aduanas)
  - `Additional` (Misceláneos)
  - `InlandFreight` (Transporte Terrestre)
</ParamField>

<ParamField body="masterId" type="integer">
  Identificador del master asociado al gasto.

  - Debe estar presente `operationId`/`operationCategory` o `masterId`; al menos uno de los dos es requerido.
</ParamField>

<ParamField body="forUseOn" type="string" required>
  Indica el uso del cargo. Valores aceptados:

  - `Invoice`
  - `Reimbursement`
</ParamField>

### Example request

<CodeGroup>

```bash curl
curl --request POST \
  --url "https://api.primenetcloud.com/v1/operation-expenses" \
  --header "X-Api-Key: YOUR_API_KEY" \
  --header "X-Tenant-Id: TENANT_ID" \
  --header "Content-Type: application/json" \
  --data '{
    "chargeId": 5,
    "chargeDescription": "Ocean Freight",
    "chargeTypeId": 1,
    "quotationGroupId": 3,
    "currencyId": "USD",
    "quantity": 1,
    "price": 1200,
    "taxPercent": 0,
    "discount": 0,
    "vendorId": 12,
    "operationId": 5321,
    "operationCategory": "Shipment",
    "forUseOn": "Invoice"
  }'
```

</ResponseField>

</ResponseField>
