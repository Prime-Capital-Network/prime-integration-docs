---
title: "Actualizar Parcial"
---

Actualiza los datos de un reembolso por pagar en la empresa.

```http
PATCH /reimbursement-liabilities/{id}
```

<Note>
  Este endpoint requiere el scope **`ReimbursementLiabilities.Write`**.
</Note>

### Path parameters

<ParamField path="id" type="integer" required>
  Identificador único del reembolso a actualizar.
</ParamField>

### Body parameters

<ParamField body="number" type="string">
  Número del reembolso. Máximo 30 caracteres.

  - Si la [configuración de CxP](/api-reference/account-payable-config/listar-config) tiene el campo `duplicatedNumbers` como `Deny`, y el número ya existe, la solicitud será rechazada con un error `400 Bad Request`.
</ParamField>

<ParamField body="date" type="date">
  Fecha del reembolso en formato `YYYY-MM-DD`.

  - Debe ser menor o igual a la fecha de hoy.
  - Debe pertenecer a un período fiscal activo.
</ParamField>

<ParamField body="vendorId" type="integer">
  Identificador del proveedor (Vendor).

  - Debe ser un asociado con el campo `IsVendor` como true.
</ParamField>

<ParamField body="identityDocumentId" type="integer">
  Identificador del tipo de documento de identidad.
</ParamField>

<ParamField body="identityDocumentNumber" type="string">
  Número de documento de identidad. Máximo 24 caracteres.
</ParamField>

<ParamField body="operationCategory" type="string">
  Categoría de la operación logística asociada. Requerido si `operationId` está presente. Valores aceptados:

  - `Shipment` (Embarque)
  - `CustomManagement` (Aduanas)
  - `Additional` (Misceláneos)
  - `InlandFreight` (Transporte Terrestre)
</ParamField>

<ParamField body="operationId" type="integer">
  Identificador de la operación logística asociada. Requerido si `operationCategory` está presente.
</ParamField>

<ParamField body="masterId" type="integer">
  Identificador del registro maestro (Master).
</ParamField>

<ParamField body="paymentTermId" type="integer">
  Identificador del término de pago.
</ParamField>

<ParamField body="reference" type="string">
  Referencia del reembolso. Máximo 250 caracteres.
</ParamField>

<ParamField body="description" type="string">
  Descripción general del reembolso. Máximo 4000 caracteres.
</ParamField>

<ParamField body="currencyId" type="string">
  Identificador de la moneda.
</ParamField>

<ParamField body="exchangeRate" type="decimal">
  Tasa de cambio aplicada. Debe ser mayor que cero.

  Si el campo es `null`:
  - Si `currencyId` es distinto a la moneda de la empresa, el sistema asignará automáticamente la tasa de cambio vigente.
  - Si `currencyId` es igual a la moneda de la empresa, se asignará por defecto el valor `1`.
</ParamField>

<ParamField body="status" type="string">
  Estatus del reembolso.

  Valores aceptados:

  - `New`
  - `Posted`
  - `Cancelled`
  - `Deleted`
</ParamField>

### Example request

<CodeGroup>

```bash curl
curl --request PATCH \
  --url "https://api.primenetcloud.com/v1/reimbursement-liabilities/1054" \
  --header "X-Api-Key: YOUR_API_KEY" \
  --header "X-Tenant-Id: TENANT_ID" \
  --header "Content-Type: application/merge-patch+json" \
  --data '{
    "number": "REI-00123",
    "date": "2026-07-13",
    "vendorId": 142,
    "paymentTermId": 2,
    "currencyId": "USD",
    "exchangeRate": 58.50,
    "operationId": 5321,
    "operationCategory": "Shipment"
  }'
```

</CodeGroup>

### Response

Si la actualización es exitosa, el endpoint retorna `204 No Content` sin body.

### Errores

<ResponseField name="404 Not Found">
  No existe ningún reembolso con el `id` especificado en la URL.
</ResponseField>

<ResponseField name="403 Forbidden">
  El API Key no tiene el scope `ReimbursementLiabilities.Write` requerido.
</ResponseField>

<ResponseField name="415 Unsupported Media Type">
  La solicitud no incluye el header `Content-Type` como `application/merge-patch+json`.
</ResponseField>
